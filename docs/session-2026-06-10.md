#### API Endpoints mới

```
GET    /api/v1/operations/assignments?eventId=X          → admin lấy tất cả phiếu của event
GET    /api/v1/operations/assignments/inbox?clubId=X     → club lấy inbox (Status != Done)
POST   /api/v1/operations/assignments?eventId=X&clubId=Y → tạo phiếu (multipart/form-data)
PATCH  /api/v1/operations/assignments/{id}/status        → cập nhật trạng thái
DELETE /api/v1/operations/assignments/{id}               → xóa phiếu
```

Sau `POST`, controller tự động notify tất cả CLUB_ADMIN của club nhận phiếu qua `INotificationService`.

## Luồng hoạt động hoàn chỉnh (4 tầng)

```
SUPER_ADMIN
  → Tạo ClubEvent (ClubId = null) tại /admin/events
  → Tạo EventClubAssignment cho từng CLB (có thể đính kèm file PDF/DOCX/...)
  → CLUB_ADMIN nhận notification "Phiếu giao việc mới"

CLUB_ADMIN
  → Vào /clubs/:id/manage/inbox
  → Đọc phiếu, xem sự kiện liên quan tại /events/university/:id
  → Nhấn "Tạo Task" → điền bảng (nhiều dòng, mỗi dòng về 1 ban, có đính kèm file)
  → Nhấn "Gửi về Ban" → tạo ClubTask thật (DepartmentId set)
  → DEPT_LEAD nhận notification "Công việc mới được giao về Ban"

DEPT_LEAD
  → Thấy task trong Backlog của ban
  → Có thể xem chi tiết event tại /events/university/:id (read-only)
  → Tạo sub-task → gán cho member → chạy trên Kanban

MEMBER
  → Nhận task trên Kanban / My Tasks
```

## Kiến trúc quan trọng

- `EventClubAssignment` là entity riêng, **không phải** `ClubTask` → backlog/kanban sạch
- File upload dùng Cloudinary qua `IFileStorageService.UploadAsync(file, folder)` (pattern đã có)
- Notify CLUB_ADMIN khi tạo phiếu: query `ClubMemberships WHERE ClubRole = CLUB_ADMIN AND ClubId = X AND Status = Active`
- Notify DEPT_LEAD khi tạo task về ban: query `ClubMemberships WHERE ClubRole = DEPT_LEAD AND DepartmentId = X AND ClubId = Y AND Status = Active`
- Cross-module notification (Membership → Operations) đặt ở Controller layer (Server project) — không vi phạm kiến trúc module isolation
- Route `/events/university/:id` mở cho mọi user đã đăng nhập; `/admin/events` vẫn chỉ dành SUPER_ADMIN
